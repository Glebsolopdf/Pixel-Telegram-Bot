# Pixel Utils Bot - Telegram Chat Manager

## Возможности

- 🔨 **Система модерации** - бан, мут, варн, кик с настраиваемыми рангами и гранулярными правами
  - Временные и постоянные баны/муты
  - Система предупреждений с автоматическими наказаниями
  - История всех наказаний
  - Автоматическое снятие истекших наказаний
  
- 👑 **Система прав рангов** - детальная настройка прав для каждого ранга модерации
  - 4 ранга модерации (Владелец, Администратор, Старший модератор, Младший модератор)
  - Гранулярные права для каждого ранга
  - Настройка прав через `/rankconfig`
  
- 📊 **Статистика чата** - отслеживание активности участников
  - Ежедневная статистика сообщений
  - Топ активных пользователей за день и за 60 дней
  - Персональные профили с графиками активности
  - Автоматический сброс статистики в полночь (МСК)
  
- 🛡️ **Защита от рейдов** - автоматическое обнаружение и блокировка спама
  - Обнаружение спама GIF-анимациями
  - Обнаружение спама стикерами
  - Обнаружение дублирующихся сообщений
  - Защита от массового вступления (рейды)
  - Настраиваемые лимиты и временные окна
  
- 🎬 **Гифки для команд** - настраиваемые анимации для действий модерации
  - Поддержка GIF для команд: ban, kick, mute, unmute, warn, unwarn, unban, moderatorslist, welcome
  - Случайный выбор GIF из папки команды
  - Включение/выключение для каждого чата отдельно
  
- 🏆 **Топ чатов** - рейтинг самых активных чатов
  - Публичный рейтинг чатов по активности
  - Настройка видимости (всегда, только публичные, никогда)
  
- 👥 **Система репутации** - оценка участников
  - Автоматическое восстановление репутации (+1 каждые 4 часа, +2 по выходным)
  - Влияние наказаний на репутацию
  - Просмотр репутации пользователей
  
- 🔗 **Сеть чатов** - интеграция между чатами с модерацией и синхронизацией настроек
  - Объединение до 5 чатов в сеть
  - Полная синхронизация настроек между чатами
  - Централизованное управление
  - Модерация чатов в сетке (закрытие/открытие, управление медиа)
  
- ⚙️ **Гибкие настройки** - персонализация под каждый чат
  - Централизованное меню настроек через `/settings`
  - Настройка системы варнов
  - Настройка защиты от рейдов
  - Настройка утилит (эмодзи-спам, спам реакциями, ложные команды)
  - Русский префикс команд
  
- 🛡️ **Защита от спама командами** - автоматическая блокировка спама
  - Middleware для обнаружения спама командами
  - Автоматический кулдаун при спаме
  - Защита настроек от несанкционированного доступа
  
- 🇷🇺 **Русские команды** - поддержка русских алиасов
  - Полная поддержка русских команд (стата, топ, профиль, мут, бан и т.д.)
  - Настраиваемый префикс для русских команд
  
- 💬 **Шепот (Whisper)** - приватные сообщения через inline-режим
  - Отправка приватных сообщений любому пользователю
  - Сообщения скрыты до нажатия кнопки просмотра
  - Автоматическое истечение через 3 часа
  - Защита от спама просмотров (кулдаун 5 секунд)

## Установка и запуск

### Требования

- **Python 3.10 или выше**
- **Telegram Bot Token** от [@BotFather](https://t.me/BotFather)

### Зависимости

Основные зависимости (см. `requirements.txt`):
- `aiogram>=3.22.0` - асинхронный фреймворк для Telegram Bot API
- `Pillow>=10.4.0` - работа с изображениями (для генерации графиков статистики)
- `python-dotenv>=1.0.0` - загрузка переменных окружения из `.env` файла

Установка всех зависимостей:
```bash
pip install -r requirements.txt
```

### Быстрый старт

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repository-url>
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройте конфигурацию:**

   **Вариант 1: Использование переменных окружения (рекомендуется)**
   
   Создайте файл `.env` в корне проекта:
   ```bash
   # Обязательные параметры
   BOT_TOKEN=your_bot_token_here
   
   # Опциональные параметры
   DEBUG=false                    # Режим отладки (true/false, по умолчанию false)
   BASE_PATH=/path/to/project     # Базовый путь к проекту (опционально)
   CLEANUP_PYCACHE_ON_SHUTDOWN=true  # Удаление __pycache__ при выключении (true/false, по умолчанию true)
   ```

   **Вариант 2: Использование config.py (не рекомендуется)**
   
   Вы можете напрямую редактировать `config.py`, но рекомендуется использовать переменные окружения через `.env` файл.

### Переменные окружения

- **BOT_TOKEN** (обязательно) - Токен бота от @BotFather
- **DEBUG** (опционально, по умолчанию `false`) - Включает режим отладки с подробным логированием
- **BASE_PATH** (опционально) - Абсолютный путь к директории проекта. По умолчанию определяется автоматически
- **CLEANUP_PYCACHE_ON_SHUTDOWN** (опционально, по умолчанию `true`) - Автоматическое удаление директорий `__pycache__` при выключении бота

### Настройки защиты от рейдов (по умолчанию)

Настройки по умолчанию можно изменить в `config.py`:

```python
RAID_PROTECTION = {
    'gif_limit': 3,              # Лимит GIF за временное окно
    'gif_time_window': 5,        # Временное окно для GIF (секунды)
    'sticker_limit': 5,           # Лимит стикеров за временное окно
    'sticker_time_window': 10,    # Временное окно для стикеров (секунды)
    'duplicate_text_limit': 3,    # Лимит похожих сообщений
    'duplicate_text_window': 30,  # Временное окно для текста (секунды)
    'mass_join_limit': 10,        # Лимит новых участников
    'mass_join_window': 60,       # Временное окно для массового вступления (секунды)
    'similarity_threshold': 0.7   # Порог схожести текста (0-1)
}
```

4. **Запустите бота:**
   ```bash
   python bot.py
   ```

## Архитектура

### Общая структура

Бот построен на базе **aiogram 3.x** и использует модульную архитектуру:

```
┌─────────────────────────────────────────────────────────┐
│                    bot.py (Entry Point)                 │
│  - Инициализация бота и диспетчера                      │
│  - Регистрация всех обработчиков                        │
│  - Запуск планировщика задач                            │
└─────────────────────────────────────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
        ▼                 ▼                 ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  Handlers    │  │  Middleware  │  │  Databases   │
│              │  │              │  │              │
│ - common     │  │ - auto_spam  │  │ - main       │
│ - moderation │  │ - command_   │  │ - moderation │
│ - settings   │  │   spam       │  │ - reputation │
│ - network    │  │ - settings_  │  │ - network    │
│ - ...        │  │   guard      │  │ - raid_      │
└──────────────┘  └──────────────┘  │   protection │
                                    │ - utilities  │
                                    └──────────────┘
```

### Middleware Chain

Middleware обрабатывает запросы в следующем порядке:

1. **AutoSpamDetectionMiddleware** - автоматическое обнаружение спама командами
   - Блокирует пользователей, отправляющих 3+ команды за 60 секунд
   - Устанавливает кулдаун на 3 минуты
   - Исключения для команд модерации

2. **CommandSpamMiddleware** - защита от спама одной и той же командой
   - Кулдаун 10 секунд между использованием одной команды
   - Исключения для модераторов

3. **SettingsGuardMiddleware** - защита настроек от несанкционированного доступа
   - Проверяет права доступа к панелям настроек
   - Разрешает доступ только владельцам и администраторам

### Система прав

Бот использует двухуровневую систему прав:

1. **Ранги модерации** (1-4) - определяют базовые права пользователя
2. **Гранулярные права** - детальная настройка для каждого ранга через `/rankconfig`

Права проверяются через `utils/permissions.py`:
- `get_effective_rank()` - получение ранга пользователя
- `check_permission()` - проверка конкретного права
- Владелец чата (creator) автоматически получает ранг 1

### Планировщик задач

Автоматические задачи выполняются в фоновом режиме через `scheduler.py`:

- **Очистка дубликатов чатов** - каждые 5 минут
- **Очистка старых записей статистики** - каждый час (старше 90 дней)
- **Обновление информации о чатах** - каждую минуту
- **Проверка истечения мутов** - каждые 10-60 секунд (зависит от наличия активных мутов)
- **Проверка истечения банов** - каждые 10-60 секунд (зависит от наличия активных банов)
- **Восстановление репутации** - каждые 4 часа (+1, +2 по выходным)
- **Очистка старых записей модерации** - каждый час (старше 7 дней)
- **Очистка истекших кодов сетки** - каждые 5 минут
- **Очистка записей защиты от рейдов** - каждые 5 минут
- **Очистка неактивных пользователей и чатов** - каждые 7 дней (неактивность > 30 дней)
- **Очистка истекших команд** - каждую минуту
- **Очистка замороженных чатов** - раз в день (после 30 дней заморозки)
- **Сброс ежедневной статистики** - в полночь по МСК

### Базы данных

Бот использует 7 отдельных SQLite баз данных:

1. **pixel_bot.db** (основная база данных)
   - Информация о чатах и пользователях
   - Статистика сообщений (ежедневная и пользовательская)
   - Права рангов и настройки чатов
   - Правила чатов

2. **moderation.db** (модерация)
   - История наказаний (баны, муты)
   - Система варнов
   - Настройки варнов

3. **reputation.db** (репутация)
   - Репутация пользователей
   - История недавних наказаний (за последние 3 дня)

4. **network.db** (сети чатов)
   - Информация о сетях
   - Связи чатов в сетях
   - Коды для подключения

5. **raid_protection.db** (защита от рейдов)
   - Настройки защиты для каждого чата
   - История активности (GIF, стикеры, текст)
   - Записи о массовом вступлении
   - Удаленные сообщения

6. **utilities.db** (утилиты)
   - История команд (для защиты от спама)
   - Настройки утилит для чатов

**Особенности:**
- Автоматическая проверка целостности баз данных при запуске
- Автоматическое восстановление при обнаружении повреждений
- Заморозка данных чата при исключении бота (данные хранятся 30 дней)
- Автоматическая очистка старых данных

## Команды

### В личных сообщениях:
- `/start` - приветствие и главное меню
- `/menu` - вернуться в главное меню
- `/net` - панель управления сеткой чатов
- "➕ Добавить в чат" - добавление бота в группу

### Основные команды (в группах):
- `/help` - справка по всем командам
- `/info` - информация о чате
- `/settings` - центральное меню настроек (требуются права администратора)
- `/top` - топ 20 активных пользователей за сегодня
- `/topall` - топ пользователей за 60 дней в этом чате
- `/myprofile` - ваш профиль с графиком активности за месяц
- `/statconfig` - настройки статистики (админы)
- `/autojoin on|off` - авто-принятие заявок на вступление

### Команды модерации:
- `/ap @username 3` - назначить ранг модератора (3 = Старший модератор)
- `/ap 3` - назначить ранг (при ответе на сообщение)
- `/unap @username` - снять ранг модератора
- `/unap` - снять ранг (при ответе на сообщение)
- `/removmymod` - снять свой ранг модератора
- `/staff` - список модераторов чата
- `/mute 10 часов` - замутить (при ответе на сообщение)
- `/mute @username 10 часов` - замутить пользователя на указанное время
- `/unmute` - размутить (при ответе на сообщение)
- `/unmute @username` - размутить пользователя
- `/kick @username` - исключить из чата
- `/kick` - исключить (при ответе на сообщение)
- `/punishhistory` или `/История наказаний` - история наказаний (модераторы)
- `/userinfo` или `/ui` - информация о пользователе из Telegram API

### Система предупреждений:
- `/warn` - выдать предупреждение (при ответе)
- `/warn @username` - выдать предупреждение
- `/unwarn` - снять предупреждение (при ответе)
- `/unwarn @username` - снять предупреждение
- `/warns` - посмотреть предупреждения (при ответе)
- `/warns @username` - посмотреть предупреждения пользователя
- `/warnconfig` - настройки системы варнов (только админы)

### Баны:
- `/ban` - забанить навсегда (при ответе)
- `/ban @username` - забанить навсегда
- `/ban 1 час` - временный бан (при ответе)
- `/ban @username 1 час` - временный бан на указанное время
- `/unban` - разбанить (при ответе)
- `/unban @username` - разбанить пользователя

### Правила чата:
- `/rules` - показать правила чата (доступно всем)
- `/rules [текст]` - установить правила чата (требуются права)
- `/rules clear` - удалить правила чата (требуются права)

### Настройка прав:
- `/rankconfig` - настройка прав рангов (владелец)
- `/initperms` - инициализация прав по умолчанию (владелец)
- `/russianprefix` - настройка префикса для русских команд (владелец)
- `/resetconfig` - сброс всех настроек к значениям по умолчанию (администратор)

### Защита от рейдов:
- `/raidprotection` - показать настройки защиты от рейдов

### Репутация:
- `/reputation` или `/rep` - показать свою репутацию
- `/reputation @username` - показать репутацию пользователя
- `/reputation` - показать репутацию (при ответе на сообщение)

### Упоминания в топах:
- `/mentionping` - включить кликабельные упоминания (ping) в топах и статистике
- `/unmentionping` - выключить кликабельные упоминания в топах и статистике

### Сетка чатов:
- `/net` - панель управления сеткой чатов (только ЛС)
- `/netconnect <код>` - подключить чат к сетке (4-значный код)
- `/netadd <код>` - добавить чат в существующую сетку (2-значный код)
- `/chatnet` - информация о сетке чатов
- `/chatnet update` - обновить информацию о чатах
- `/unnet` - отключить чат от сетки

### Шепот (Whisper):
- Отправка через inline-режим: `@бот ваш текст @получатель`
- Шепоты автоматически истекают через 3 часа
- Максимальная длина сообщения: 1000 символов

### Ранги модерации:
- **1** - Владелец 👑 (все права)
- **2** - Администратор ⚜️ (настройки и назначение рангов)
- **3** - Старший модератор 🛡 (баны и кики)
- **4** - Младший модератор 🔰 (варны и муты)

### 🇷🇺 Русские команды:

Бот поддерживает русские алиасы для основных команд:

**Статистика:**
- `стата` → `/top`
- `топ` → `/top`
- `стата вся` → `/topall`
- `статистика вся` → `/topall`

**Профиль:**
- `профиль` → `/myprofile`
- `мой профиль` → `/myprofile`
- `кто ты` → `/myprofile`

**Настройки:**
- `настройки` → `/settings`
- `конфиг` → `/settings`
- `правила` → `/rules`
- `правила очистить` или `очистить` → `/rules clear`
- `автодопуск` → `/autojoin`

**Модерация:**
- `мут` → `/mute`
- `размут` → `/unmute`
- `кик` → `/kick`
- `бан` → `/ban`
- `разбан` → `/unban`
- `варн` → `/warn`
- `разварн` → `/unwarn`
- `назначить` → `/ap`
- `снять` → `/unap`

**Служебное:**
- `кто админ` → `/staff`
- `история наказаний` → `/punishhistory`
- `антирейд` → `/raidprotection`
- `настройки антирейд` → `/raidprotection`

## Настройка

Большинство настроек доступны через команду `/settings` в группе. Для доступа к настройкам требуются права администратора или владельца чата.

### Система прав рангов

Бот поддерживает детальную настройку прав для каждого ранга модерации. Вы можете настроить:
- Права на модерацию (бан, мут, кик, варн)
- Права на назначение рангов
- Права на доступ к настройкам
- Права на просмотр статистики и истории наказаний
- Права на управление правилами

**Ранги модерации:**
- **1 - Владелец 👑** - все права, включая настройку прав других рангов
- **2 - Администратор ⚜️** - настройки и назначение рангов 3-4
- **3 - Старший модератор 🛡** - баны, кики, муты, варны
- **4 - Младший модератор 🔰** - только варны и муты

**Настройка прав:**
- Используйте `/rankconfig` для детальной настройки прав каждого ранга
- Используйте `/initperms` для восстановления прав по умолчанию

### Сеть чатов

Сеть чатов позволяет связать до 5 чатов для:
- Просмотра общей статистики
- Синхронизации всех настроек между чатами
- Централизованного управления
- Модерации чатов (закрытие/открытие, управление медиа)
- Удаления сетки

**Создание сети:**
1. Используйте `/net` в личных сообщениях с ботом
2. Выберите "Создать сетку" - получите 4-значный код
3. В другом чате используйте `/netconnect <код>` для подключения
4. Для добавления дополнительных чатов используйте `/netadd <2-значный код>`

**Синхронизация настроек:**
При синхронизации из исходного чата копируются все настройки:
- Настройки варнов (лимит, тип наказания, длительность мута)
- Настройки статистики (включена/выключена)
- Права рангов (все права для всех рангов модерации)
- Русский префикс команд
- Автодопуск (включен/выключен, уведомления)
- Настройки анти-спама (все параметры защиты от рейдов)
- Настройки утилит (эмодзи-спам, спам реакциями, ложные команды)
- Настройки гифок (включены/выключены)
- Настройки топ чатов (отображение в топе)

**Функции модерации в сетке:**
- Закрытие/открытие чата (отключение/включение отправки сообщений)
- Управление медиа (отключение/включение отправки медиа-файлов)
- Удаление сетки с подтверждением
- Просмотр общей статистики по всем чатам сети

**Коды подключения:**
- 4-значный код - для создания новой сети (первый чат)
- 2-значный код - для добавления чата в существующую сетку
- Коды автоматически истекают через определенное время

### Система гифок

Бот поддерживает настраиваемые GIF-анимации для команд модерации:

**Поддерживаемые команды:**
- `ban` - при бане пользователя
- `kick` - при кике пользователя
- `mute` - при муте пользователя
- `unmute` - при размуте пользователя
- `warn` - при выдаче варна
- `unwarn` - при снятии варна
- `unban` - при разбане пользователя
- `moderatorslist` - при просмотре списка модераторов
- `welcome` - при приветствии нового участника

**Как это работает:**
- Поместите GIF-файлы в соответствующие папки в `Gifs/`
- Бот случайным образом выбирает один GIF из папки команды
- Можно включить/выключить гифки для каждого чата отдельно
- Настройка через `/settings` → Гифки

**Формат файлов:**
- Поддерживаются файлы `.mp4`, `.MOV` и другие видеоформаты
- Рекомендуется использовать оптимизированные файлы для быстрой загрузки

### Система репутации

Система репутации оценивает участников на основе их поведения:

**Начальная репутация:** 100

**Восстановление:**
- +1 каждые 4 часа (если нет наказаний за последние 24 часа)
- +2 каждые 4 часа по выходным (суббота, воскресенье по МСК)

**Влияние наказаний:**
- Наказания снижают репутацию
- Недавние наказания (за последние 3 дня) учитываются при восстановлении

**Просмотр:**
- `/reputation` - своя репутация
- `/reputation @username` - репутация другого пользователя

### Топ чатов

Система рейтинга самых активных чатов:

**Настройки видимости:**
- `always` - всегда отображаться в топе
- `public_only` - только если чат публичный (по умолчанию)
- `never` - никогда не отображаться в топе

**Настройка:**
- Через `/settings` → Топ чатов
- Или через `config.py` → `TOP_CHATS_DEFAULTS`

**Критерии:**
- Рейтинг основан на активности чата (количество сообщений)

**Особенности:**
- Только один активное голосование на пользователя
- Автоматическое закрытие голосования при достижении кворума
- Уведомления о результатах голосования

### Автоматическая заморозка данных

Бот автоматически замораживает данные чата (устанавливает `is_active = 0` и `frozen_at`) в следующих случаях:
- Бот был исключен из чата (`kicked`)
- Бот покинул чат (`left`)
- Ошибка "chat not found" при попытке обновить информацию о чате

Замороженные чаты не обрабатывают callback-запросы и команды. Данные остаются в базе для возможного восстановления в течение 30 дней. После 30 дней данные автоматически удаляются.

## Шепот (Whisper)

Функция шепота позволяет отправлять приватные сообщения любому пользователю через inline-режим бота. Сообщения скрыты до момента просмотра получателем.

### Как отправить шепот

   **Через inline-режим:**
   - Откройте любой чат в Telegram
   - Начните вводить `@ваш_бот` (username бота)
   - Введите ваш текст и username получателя: `@бот ваш текст @получатель`
   - Выберите результат из списка и отправьте

### Особенности и лимиты

- **Приватность:** Сообщение скрыто до момента просмотра получателем
- **Время жизни:** Шепоты автоматически истекают через 3 часа
- **Длина сообщения:** Максимум 1000 символов
- **Защита от спама:** Кулдаун 5 секунд между просмотрами шепотов
- **Независимость:** Получателю не обязательно существовать в базе данных бота - шепот можно отправить любому пользователю Telegram по его username
- **Ограничения:**
  - Нельзя отправить шепот самому себе
  - Нельзя отправить шепот боту

### Технические детали

- Шепоты хранятся в памяти (in-memory cache)
- Автоматическая очистка истекших шепотов
- Rate limiting для просмотра (5 секунд между просмотрами)
- Поддержка длинных сообщений (отправка в чат вместо alert)

### Примеры использования

**Отправка шепота:**
```
@бот Привет! Как дела? @username
```

## Защита от рейдов

Система защиты от рейдов автоматически обнаруживает и блокирует различные виды спама:

### Обнаружение спама GIF

- **Лимит:** 3 GIF за временное окно (по умолчанию)
- **Временное окно:** 5 секунд (по умолчанию)
- **Действие:** Удаление сообщения и логирование инцидента

### Обнаружение спама стикерами

- **Лимит:** 5 стикеров за временное окно (по умолчанию)
- **Временное окно:** 10 секунд (по умолчанию)
- **Действие:** Удаление сообщения и логирование инцидента

### Обнаружение дублирующихся сообщений

- **Лимит:** 3 похожих сообщения за временное окно (по умолчанию)
- **Временное окно:** 30 секунд (по умолчанию)
- **Порог схожести:** 0.7 (70%, по умолчанию)
- **Алгоритм:** Использует SequenceMatcher для сравнения текста
- **Действие:** Удаление сообщения и логирование инцидента

### Защита от массового вступления

- **Лимит:** 10 новых участников за временное окно (по умолчанию)
- **Временное окно:** 60 секунд (по умолчанию)
- **Действие:** Логирование инцидента (можно настроить автоматический бан)

### Настройка защиты

Все параметры защиты настраиваются через `/settings` → Защита от рейдов:
- Включение/выключение защиты
- Настройка лимитов для каждого типа спама
- Настройка временных окон
- Настройка порога схожести текста
- Silent mode (без уведомлений)

## Устранение неполадок

### Проблемы с базой данных

**Ошибка: "database is locked"**
- Убедитесь, что бот не запущен в нескольких экземплярах
- Проверьте, нет ли других процессов, использующих базу данных

**Повреждение базы данных:**
- Бот автоматически проверяет целостность при запуске
- При обнаружении повреждений запускается автоматическое восстановление
- Если восстановление не удалось, проверьте логи

**Очистка базы данных:**
- Старые записи автоматически очищаются планировщиком
- Можно вручную удалить файлы в папке `data/` (будет создана новая база)

### Проблемы с правами бота

**Бот не отвечает на команды:**
- Убедитесь, что бот имеет права администратора в чате
- Проверьте, что бот не был исключен из чата
- Используйте `/info` для проверки статуса бота

**Бот не может выполнять действия модерации:**
- Бот должен иметь права администратора с соответствующими разрешениями:
  - Удаление сообщений
  - Ограничение участников (для мута)
  - Исключение участников (для кика)
  - Блокировка участников (для бана)

### Проблемы с конфигурацией

**Ошибка: "BOT_TOKEN не задан"**
- Убедитесь, что файл `.env` существует в корне проекта
- Проверьте, что переменная `BOT_TOKEN` установлена
- Убедитесь, что установлен пакет `python-dotenv` (`pip install python-dotenv`)

**Бот не запускается:**
- Проверьте версию Python (требуется 3.10+)
- Установите все зависимости: `pip install -r requirements.txt`
- Проверьте логи на наличие ошибок

### Проблемы с производительностью

**Высокое использование памяти:**
- Планировщик автоматически очищает старые данные
- Проверьте количество активных чатов
- Рассмотрите возможность очистки неактивных чатов

**Медленная работа:**
- Проверьте размер баз данных
- Запустите очистку старых данных вручную (если необходимо)
- Убедитесь, что диск не переполнен

### Восстановление после сбоя

**Восстановление замороженного чата:**
- Добавьте бота обратно в чат
- Бот автоматически активирует чат при следующем обновлении информации
- Или используйте команду `/info` для принудительного обновления

**Восстановление настроек:**
- Используйте `/resetconfig` для сброса всех настроек к значениям по умолчанию
- Или настройте вручную через `/settings`
